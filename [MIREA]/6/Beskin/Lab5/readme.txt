Программный комплекс «Властелин каналов» предназначен для эмуляции работы каналов ввода-вывода многоканальной ЭВМ.
«Властелин каналов» демонстрирует работу селекторного и мультиплексного каналов и позволяет изучить разницу в их функционировании. С помощью программы можно оценить эффективность использования того или иного канала и устройств, подключаемых к данным каналам, и проследить, как изменение конфигурации каналов и устройств влияет на эффективность работы многоканальной системы ввода-вывода. Кроме того, программный комплекс позволяет оптимизировать набор задач для эффективного использования каналов и устройств с конкретной конфигурацией.
Областью применения программы являются лабораторные работы по дисциплине «Операционные системы».




Для работы LoCH требуется наличие библиотек Microsoft .Net Framework 2.0


Программа поставляется в следующей комплектации
1. Системные фаилы:
NewLoCH.exe – исполняемый файл ВК
price.lchd- фаил настроек.
2. Файлы вариантов задач с расширением *.lchv






Описание главного окна ВК

Главное окно ВК содержит восемь кнопок:
Эмуляция
Статистика
Процессы
Каналы и устройства
Сохранить решение
Загрузить решение
Выход

Начинать работу следует с загрузки варианта ( фаила с разрешением lchv), который содержит изначальные настройки кагалов и устройств, а также таблицу выполняем процессов.
Нажатие на кнопку Эмуляция выводит на экран окно эмуляции. Эмуляция производится с учетом настроек, указанных в пунктах Процессы и  Каналы и устройства.
Если пользователь не менял настройки каналов и устройств и набор процессов в вышеуказанных пунктах, то эмуляция производится с настройками, содержащимися в программе по умолчанию.
Статистика отображает состояние процессов ПОСЛЕ ЗАВЕРШЕНИЯ ЭМУЛЯЦИИ.
Процессы, которые задействуют ресурсы каналов и устройств, содержатся в таблице процессов. Чтобы  получить информацию о наборе процессов, согласно которому будет проводиться эмуляция, следует нажать кнопку Процессы.
Для изменения настроек каналов и подключаемых к ним устройств следует нажать  кнопку Каналы и устройства.
Программа позволяет и настройки каналов и устройств ( решение), чтобы впоследствии пользователь мог загрузить ранее сохраненные настройки и производить эмуляцию с этими настройками.
Решение может быть также загружена из текстового файла,  описание его структуры смотри в файле пример_решения.txt
 






Программный комплекс «Властелин каналов» позволяет эмулировать работу 5 каналов. К каждому каналу может быть подключено до 5 устройств.

Каждый канал характеризуется:
типом (селекторный или мультиплексный)
задержкой передачи байта данных
скоростью передачи данных
устройствами, подключаемыми к каналу
Каждое устройство характеризуется:
объемом буфера
задержкой вывода байта данных
скоростью чтения.

Для изменения этих параметров необходимо нажать кнопку с номером желаемого канала. Зпатем кнопку с номером желаемого устройства. И изменить необходимые параметры в соответствующих счетчиках.

Каждое изменение имеет свою цену., отображаемую во всех окнах программы.
Необходимо стремиться к минимизации этой цены.
Окно Процессы предназначено для отображения процессов, которые используют ресурсы каналов и устройств во время эмуляции.
Процессы отображаются в окне в виде таблицы процессов. Каждый процесс имеет параметры:
номер канала, который данный процесс задействует
номер устройства, который данный процесс задействует
время запуска – время, в которое данный процесс должен начать выполняться
время окончания – время, до которого процесс должен быть выполнен (если процесс не успеет выполниться до этого времени, то он будет снят с выполнения)
объем данных, которыми обменивается данный процесс с устройством, указанным в параметре Номер устройства





Окно Эмуляция предназначено для отображения процесса эмуляции работы каналов и подключенных к ним устройств. Работа каналов и устройств обусловлена их настройками и набором процессов, задействующих данные каналы и устройства.
В окне отображаются 5 каналов, к каждому из которых подключено 5 устройств. 
Каждое устройства представлено двумя индикаторами, отображающие заполненость буферов соответствующщих процессу обмена между каналом и устройством.

В нижней части окна отображается счетчик тактов. Процесс задействует канал и устройство в тот момент времени, который записан в его параметре Время запуска.
Для начала эмуляции следует начать на кнопку соответствующую желаемой скорости эмуляции, или указать целеевой такт и нажать кнопку Перейти.
Для того, чтобы остановить эмуляцию, следует нажать кнопку Стоп.
Для сброса модели в начальное состояние необходимо нажать кнопку Сброс.
Для просмотра статистики следует нажать кнопку Статистика.
Если объем задачи меньше или равен объему буфера устройства, которое задействует данная задача, тогда данные выводятся за один сеанс. Иначе для вывода данных требуется несколько сеансов.
Селекторный канал работает с устройством до тех пор, пока все данные не будут выведены (пока весь текущий процесс не будет отработан) и только после этого приступает к передаче данных другому подключенному к нему устройству.
Мультиплексный канал работает с устройством до тех пор, пока его буфер не будет заполнен, после этого приступает к работе с другим устройством.



 Статистика

Окно статистика предназначено для отображения результатов моделирования.
Окно содержит статистику по работе каналов и подключенным к ним устройств. Для каждого канала указаны отработанные процессы, задействованные ими устройства, объем процесса, и объем и процент потерянных данных.
Данные могут быть потеряны, когда к моменту времени, указанному в параметре «время окончания процесса», процесс не успел отработать. В таком случае процесс снимается с выполнения, и часть данных не успевает выводиться.
